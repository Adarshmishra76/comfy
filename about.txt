                    AI STRESS DETECTION APPLICATION — "COMFY"

1. PROJECT OVERVIEW
-------------------
Comfy is a full-stack AI-powered stress detection and mental wellness web
application. It analyzes user responses through a dynamically generated,
personalized questionnaire and provides tailored stress management
recommendations. The application uses Google's Gemini 2.5 Flash AI model to
generate context-aware assessment questions based on the user's profile
(age, gender, occupation, mood, sleep hours, activity level) and then analyzes
their responses to deliver actionable insights, personalized tips, and a
weekly wellness plan.

The platform supports user authentication with JWT-based access/refresh token
system, assessment history tracking, stress trend analytics with cluster-level
scoring, and bilingual support (English & Hindi). It is designed as a
privacy-first, non-clinical tool for stress awareness — not medical diagnosis.

Live URL: https://mycomfyy.netlify.app
Backend:  https://comfy-o2ia.onrender.com

4. BACKEND ARCHITECTURE (MVC PATTERN)
================================================================================

The server follows the MVC (Model-View-Controller) architectural pattern with
a service layer for AI logic:

  Request → Route → Middleware → Controller → Service → Model → Database

4.1 MIDDLEWARE STACK (Order of execution in server.js):
  1. CORS          — Whitelisted origins (Netlify + localhost), credentials: true
  2. Helmet        — Security headers (XSS, MIME sniffing, clickjacking)
  3. Morgan        — HTTP request logging (piped to Winston logger)
  4. express.json  — JSON request body parsing
  5. cookie-parser — Parses HTTP-only cookies (refresh tokens)
  6. Rate Limiter  — 100 requests / 15 min per IP (skipped in dev mode)

4.2 GRACEFUL SHUTDOWN:
  The server handles SIGINT and SIGTERM signals to:
  - Close the HTTP server
  - Close the MongoDB connection
  - Exit the process cleanly

5. API ENDPOINTS
================================================================================

5.1 AUTHENTICATION ROUTES (/api/auth)
---------------------------------------
POST   /api/auth/register     — Register new user (rate-limited)
  Request:  { name, email, password }
  Response: { user, accessToken } + refreshToken cookie

POST   /api/auth/login         — Login user (rate-limited)
  Request:  { email, password }
  Response: { user, accessToken } + refreshToken cookie

GET    /api/auth/logout        — Logout user (Protected)
  Response: Clears refreshToken cookie

GET    /api/auth/me            — Get current user profile (Protected)
  Response: { user object }

POST   /api/auth/refresh       — Refresh access token (Public)
  Reads:    refreshToken from HTTP-only cookie
  Response: { accessToken }

5.2 ASSESSMENT ROUTES (/api)
------------------------------
POST   /api/generate-questions — Generate AI questions (Optional Auth)
  Request:  { userInfo: { age, gender, occupation, mood, language, ... } }
  Response: { questions: [{ id, cluster, text, minLabel, maxLabel }] }
  Details:  AI generates 10 personalized questions across 5 clusters.
            Falls back to curated pool if AI fails.
            Logged-in users have daily AI request limits (100/day).

POST   /api/analyze-stress     — Analyze responses & get results (Optional Auth)
  Request:  { userInfo, answers: { questionId: value }, questions }
  Response: {
    score (0-100), level, analysis, personalizedTips[5],
    wellnessPlan, clusterScores, disclaimer
  }
  Details:  Score calculated server-side. AI provides personalized analysis.
            Results saved to MongoDB for logged-in users.
            Falls back to curated tips on AI failure.

GET    /api/history             — Get past assessments (Protected)
  Response: Array of last 10 assessments (sorted by date)

5.3 ANALYTICS ROUTES (/api)
------------------------------
GET    /api/trends              — Get user stress trends (Protected)
  Response: { last7DaysAvg, last30DaysAvg, changePercentage,
              dominantCluster, totalAssessments, history[] }

GET    /api/admin/analytics     — Platform-wide analytics (Admin only)
  Response: { totalAssessments, totalUsers, platformAvgScore }

5.4 USER ROUTES (/api/user)
------------------------------
GET    /api/user/me             — Get user profile (Protected)

5.5 HEALTH CHECK
-----------------
GET    /ping                    — Server health check
  Response: { status: "Active", timestamp }

6. DATABASE SCHEMA (MongoDB + Mongoose)
================================================================================

6.1 USERS COLLECTION
---------------------
{
  name:           String (required, max 50 chars),
  email:          String (required, unique, regex validated),
  password:       String (required, min 6 chars, bcrypt hashed, select: false),
  role:           String (enum: ['user', 'admin'], default: 'user'),
  aiRequestCount: {
    date:  String (YYYY-MM-DD, resets daily),
    count: Number (default: 0, max: 100/day)
  },
  createdAt:      Date (auto)
}

Pre-save Hook: Encrypts password with bcrypt (10 salt rounds).
Instance Method: matchPassword() — compares entered password with hash.

6.2 ASSESSMENTS COLLECTION
----------------------------
{
  userId:          ObjectId (ref: User, required, indexed),
  userInfo:        { age, gender, occupation, mood },
  questions:       [{ id, text, cluster, answer (1-5) }],
  score:           Number (0-100),
  clusterScores:   { Work, Emotional, Physical, Social },
  level:           String (enum: Low/Moderate/High),
  analysis:        { summary, keyStressors[], stressLevelExplanation },
  personalizedTips:[{ title, description }],
  createdAt:       Date (auto)
}

Compound Index: { userId: 1, createdAt: -1 } — optimizes user history queries.

7. AUTHENTICATION SYSTEM
================================================================================

7.1 TOKEN STRATEGY (Dual JWT):
  - Access Token:  Short-lived (15 minutes), sent in Authorization header
  - Refresh Token: Long-lived (7 days), sent as HTTP-only secure cookie

7.2 COOKIE CONFIGURATION:
  - httpOnly: true      — Not accessible via JavaScript (XSS protection)
  - secure: true        — Only sent over HTTPS
  - sameSite: 'none'    — Required for cross-site (Netlify → Render)
  - expires: 7 days

7.3 TOKEN REFRESH FLOW (Client-side):
  1. Axios interceptor detects 401 response
  2. Automatically calls POST /api/auth/refresh
  3. Gets new access token from server
  4. Retries the original failed request
  5. If refresh fails → user is logged out

7.4 SESSION VERIFICATION ON APP LOAD:
  - AuthContext calls /api/auth/refresh on mount
  - If valid refresh cookie exists → restores session
  - If no cookie → user stays logged out (no error)

7.5 AUTH MIDDLEWARE TYPES:
  - protect:      Blocks request if no valid token
  - authorize:    Role-based access control (user/admin)
  - optionalAuth: Attaches user if token exists, never blocks

8. AI SERVICE IMPLEMENTATION
================================================================================

8.1 MODEL CONFIGURATION:
  - Model: gemini-2.5-flash
  - Temperature: 1.0 (high creativity for variety)
  - Top P: 0.95, Top K: 40
  - Max Output Tokens: 8192
  - Response MIME Type: application/json (structured output)

8.2 QUESTION GENERATION (10 questions, 5 clusters):
  Input:  User profile (age, gender, occupation, mood, sleep, activity)
  Output: 10 questions distributed across 5 stress clusters:
    - Work/Academic Pressure (2 questions)
    - Emotional Well-being (2 questions)
    - Physical & Sleep Health (2 questions)
    - Social & Lifestyle Balance (2 questions)
    - Lifestyle & Habits (2 questions)

  Variety Mechanisms:
    - Random session seed (unique per call)
    - Random question angle (10 themes: time management, decision fatigue, etc.)
    - Random style mix (scenario-based, frequency, behavioral, etc.)
    - Randomized cluster emphasis (2 clusters prioritized per session)
    - Occupation classification (Student/Professional/Freelancer/Other)
    - Role-specific prompt coaching per occupation type

8.3 STRESS ANALYSIS:
  Input:  User profile + answers with intensity labels + calculated score
  Output: Personalized analysis with:
    - Score interpretation in user's life context
    - 2-4 specific key stressors from high-scoring answers
    - 5 personalized actionable tips
    - Weekly wellness plan (3 measurable goals)
    - Recommended technique from curated list
    - Disclaimer

  Variety Mechanisms:
    - Random tip style (habit-building, mindset-shift, quick win, etc.)
    - Random tip angle (morning routine, evening, digital habits, etc.)
    - Random summary tone (warm, direct, growth mindset, practical)
    - Banned themes pool (rotated each session to prevent repetition)
    - 18 wellness techniques to choose from

8.4 FALLBACK SYSTEM:
  If AI fails (rate limit, API down, key missing):
    - Questions: Returns shuffled subset from 12-question curated pool
    - Analysis: Returns occupation-aware tips from randomized fallback pool
    - Both support English and Hindi

8.5 RETRY LOGIC:
  - Max retries: 3
  - Exponential backoff: 2^(attempt) seconds + random jitter
  - Only retries on 429 (rate limit) or quota errors

8.6 SCORE CALCULATION (Server-side, deterministic):
  Overall: (sum of all answers) / (number of questions × 5) × 100
  Cluster: (cluster sum) / (cluster count × 5) × 100
  Level:   0-30 = Low, 31-60 = Moderate, 61-100 = High

================================================================================

  - AI Unavailable:    Falls back to curated question & tip pools
  - Rate Limited:      Exponential backoff retry (3 attempts + jitter)
  - Database Down:     App continues; history/saving disabled
  - Token Expired:     Auto-refresh via Axios interceptor
  - Refresh Failed:    User logged out gracefully
  - API Errors:        Standardized response format with error details
  - 404 Routes:        JSON error from backend
  - Global Handler:    Catches all unhandled errors in errorHandler.js
  - Logging:           Winston logs errors to file + console
